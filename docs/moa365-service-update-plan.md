# moa365 브랜드 → 서비스 반영 마스터 가이드

> 근거: `moa365-brand-playbook.md` + 디자인 레퍼런스(`docs/moa365-color-system.png`, `docs/moa365-graphics.png`).
> 이 문서는 플레이북을 **실행 가능한 작업 항목**으로 분해한 엔지니어링/콘텐츠 가이드다. 작성: 2026-06.

## 0. 대원칙 (모든 표면 검수 게이트 — 플레이북 부록)

1. 행동을 **시키는** 문안인가? → 그렇다면 **평어**(메타포 금지).
2. 행동에 **의미를 돌려주는** 문안인가? → 표면 세계관 단어는 **'여정' 하나만** 허용.
3. **점-선 모티프**가 1회 이상 들어갔는가?
4. **AI 스멜**(가짜 인물·과장 후기) 없는가?
5. 에피소드 vs **서사** — 시간의 축적을 보여주는가?

내부 세계관(항해·선장·항해일지)은 기획 관점 전용. **앱/스토어/광고 표면 노출 금지.**

## 2. 디자인 시스템 (코드: `docs/DESIGN_SYSTEM.md` 참조)

- **4색 시맨틱(제품 기능 축 — 지출 카테고리 10색과 별개)**: Record 오렌지→앰버 `#FF6B2C→#FFA63D` / Journey 시안→블루 `#01DCE3→#2D7DFF` / Insight 바이올렛→퍼플 `#7B61FF→#A78BFA` / Growth 라임→그린 `#9BE15D→#00C48C`. 그라데이션 방향 좌→우 통일(=여정).
- **점-선 모티프**: 점=하루 기록, 선=여정. 재사용 컴포넌트 `frontend/src/components/brand/DotLine.tsx`.

- 토큰: `tailwind.config.js`(`record/journey/insight/growth` 색 + `bg-grad-*`), `index.css`(Pretendard @import + font-family).
