# moa365 마케팅 업무 R&R — 기획·제작 역할 편성

> 근거: `moa365-brand-playbook.md` Part 5(영상)·6(캠페인/콘텐츠)·9(인지·가입). 엔지니어링 R&R(`infra/docs/AGENT_TEAM.md` §7)과 **분리·병행** 운영. 서비스 작업 가이드: `docs/moa365-service-update-plan.md`.

## 0. 최상위 운영 원칙 (모든 마케팅 산출물)

- **언어 레이어 분리**: 퍼포먼스 소재 = **L1 언어 100%**(메타포 0%, 기능 평어). 브랜드/캠페인 소재 = **L2~L3**(세계관 표면 단어는 '여정' 하나만).
- **내부 세계관(항해·선장·항해일지) 표면 노출 금지** — 영상 내레이션/About 페이지에서만 제한적 허용.
- **점-선 모티프 1회 이상** + **4색 시맨틱**(Record/Journey/Insight/Growth) 일관 적용(상기력 축적).
- **AI 스멜 금지**: 가짜 인물·과장 후기 금지. 실제 기록 화면·실제 사용 장면·손맛 요소(손글씨 한 줄·영수증 질감) 우선.
- 산출물 검수는 **플레이북 부록 5문항 체크리스트** 통과 필수(§7).

## 4. 단기 실행 우선순위 (인지도 0 → 초기 1,000 유저)

1. **스토어 카피 교체 + 온보딩 문안 적용** — 비용 0, 즉시 (M2·R8) ※ 서비스 PR과 동기화
2. **데모 쇼츠 Ver.C(무자막 3초 데모)** 제작·게시 — 제작비 최소·설치효율 최대 (M4·M2)
3. **블로그 SEO 3편** — 검색 유입은 복리 (M5)
4. **#하루한줄 챌린지 1기** — ATM 팔로워(YouTube 8.6K + Threads 2.5K) 대상 클로즈드 베타 겸 (M3) ※ 베타 테스터 모집과 통합
5. 브랜드 필름은 **M2 이후** 제작 — 퍼널이 먼저, 아우라는 그다음 (M4·M1)

## 5. 퍼널 & 측정 (M6 오너)

| 단계 | 장치 | 핵심 지표 |
|---|---|---|
| 인지 | 쇼츠 3종 + Threads + 블로그 | 도달·조회 |
| 방문 | 랜딩 히어로(기능 슬로건)+3초 데모 | 방문→가입률 |
| 가입 | OAuth 1탭 → 온보딩 직행 | 가입 수 |
| 활성 | 가입 60초 내 첫 줄 | **D0 기록률(최우선)** |
| 습관 | 30일 챌린지+푸시+축적 피드백 | D7/D30 리텐션 |
| 전환 | 월간 리뷰 블러+30일 카드 쿠폰 | 프리미엄 전환율 |
| 확산 | 결산카드 공유+추천코드 | K-factor |

- **브랜드 강도**: 분기 1회 "한 줄 가계부 하면 떠오르는 앱?" 서베이(Threads) — 컨셉↔이름 연결도 추적.
- 모든 캠페인 **UTM 분리**, 결산카드 유입은 **전용 딥링크**.

## 6. 엔지니어링 ↔ 마케팅 핸드오프

- **공용 자산**: 점-선 모티프(`DotLine`)·4색 시맨틱·Pretendard — `docs/DESIGN_SYSTEM.md §14b`. 마케팅 그래픽도 동일 토큰.
- **스토어 카피**: `infra/docs/PLAY_STORE_LISTING.md` + `moa365-service-update-plan.md §6`. R8/R10 검수.
- **30일 카드 / Moa Wrapped**: 기능은 R3/R7(P2), 디자인·공유 카피는 M1/M4. **공유=광고 구조**(하단 워터마크 = moa365 로고+점선).
- **측정 연동**: 결산카드 딥링크·UTM은 M6↔R3(딥링크 라우팅) 협업.

## 7. 산출물 검수 게이트 (플레이북 부록 — 마케팅용)

1. 행동을 **시키는** 문안인가? → 평어인가(메타포 금지)?
2. 의미를 **돌려주는** 문안인가? → '여정'까지만 허용?
3. **점-선 모티프** 1회 포함?
4. **AI 스멜**(가짜 인물·과장 후기) 없는가?
5. 에피소드 vs **서사** — 시간의 축적을 보여주는가?

> 퍼포먼스 소재는 1~2를 특히 엄격히(L1 100%). 브랜드 소재는 5를 중심으로.
