## 6. 데이터 안전성 양식 (Play Console > 앱 콘텐츠 > 데이터 안전성)

각 항목에 그대로 복사해서 답변.

### Q1. 데이터 수집·공유 여부
**예** — 데이터 수집함, 외부 공유 안 함.

### Q2. 수집 데이터 종류

| 데이터 유형 | 수집함? | 목적 | 선택/필수 |
|---|---|---|---|
| **개인 식별자** | | | |
| - 이름 | ✓ | 앱 기능, 계정 관리 | 선택 |
| - 이메일 주소 | ✓ | 앱 기능 (로그인), 계정 관리 | 필수 |
| - 사용자 ID | ✓ | 앱 기능 | 필수 |
| **재무 정보** | | | |
| - 사용자 입력 지출 내역 | ✓ | 앱 기능 (가계부) | 필수 |
| - 사용자 입력 예산 / 수입 | ✓ | 앱 기능 (예산 알림) | 선택 |
| **사진/동영상** | | | |
| - 영수증 사진 (사용자 업로드) | ✓ | 앱 기능 (지출에 첨부) | 선택 |
| **앱 활동** | | | |
| - 앱 내 상호작용 | ✓ | 분석 (사용 패턴) | 선택 |

| **메시지** | ✗ | — | — |
| - 연락처 | ✗ | — | — |
| - 기기 ID | ✓ (FCM 토큰) | 앱 기능 (푸시 알림) | 필수 |

### Q3. 데이터 암호화
**예** — 전송 중 데이터는 HTTPS 로 암호화 (TLS 1.2+).
저장 데이터는 데이터베이스 수준 암호화.

### Q4. 사용자가 데이터 삭제 요청 가능?
**예** — 앱 내 마이페이지에서 계정 삭제 가능. 즉시 soft delete, 30일 후 영구 삭제.

### Q5. 데이터 공유 (제3자)
**아니오** — Moa 는 데이터를 제3자에게 공유하거나 판매하지 않음.

## 8. 출시 트랙 (Play Console > 출시)

권장 단계:

### 1단계: 내부 테스트 (Internal Testing)
- 최대 100명 (Google 계정 기반)
- 심사 없이 즉시 배포
- 본인 + 가족·친구 5-10명 초대 → 실제 사용 피드백
- 권장 기간: **3-7일**

### 2단계: 비공개 테스트 (Closed Testing)
- 이메일 또는 Google Group 으로 초대
- 심사 1-2일
- 베타 사용자 20-50명
- 권장 기간: **1-2주**

### 3단계: 공개 테스트 (Open Testing)
- Play Store 에서 누구나 찾을 수 있음 (공개 베타)
- "Open beta" 라벨 표시
- 권장 기간: **선택 — 정식 출시로 바로 갈 수도 있음**

### 4단계: 프로덕션 (Production)
- 정식 출시
- 심사 1-3일
- 모든 Play Store 사용자에게 노출
